## 11. STOP conditions

Stop and report instead of proceeding normally when:

- the user wrote `{content}` alone and no analyzable content is available;
- both GitHub source and live files are missing/unreadable and no fallback instruction exists;
- source/live conflict creates unsafe ambiguity;
- the task would require changing the four-output structure without user consent;
- the task would silently obsolete a rule without authority;
- the requested action would overclaim repo verification;
- the requested Claude prompt would require unsafe writes without proper governance constraints;
- the requested response would overclaim local repo state, tests, commits, or hashes.

A STOP report must still use the four-output structure when the `{content}` workflow is active.
